# 0046. 사용하지 않는 시리즈 기능을 폐기한다

Date: 2026-07-21
Status: Accepted
Supersedes: 0003, 0011, 0036의 시리즈 기능 관련 결정

## 배경

Ark에서 시리즈 콘텐츠를 더 이상 운영하지 않는다. 그러나 Engineering 시리즈
라우트, 시리즈별 사이트맵 항목, 글 메타데이터 스키마, 상세 글 내비게이션,
카드 표시, 품질 감사 규칙이 남아 있어 사용하지 않는 경로와 유지 비용을
계속 만들고 있었다.

## 결정

- `/engineering/series/[seriesId]`와 `/series` redirect를 제거한다.
- 시리즈 그룹 모델, 상세 글 시리즈 내비게이션, 카드의 시리즈 표시를 제거한다.
- `FeedFrontmatter`와 Zod schema에서 `series` 필드를 제거한다.
- 사이트맵과 Engineering 목록은 개별 공개 글만 다룬다.
- 기존 ADR은 당시의 콘텐츠 구조와 탐색 결정을 기록한 역사로 보존하고, 현재
  구현 기준은 이 ADR을 따른다.

## 결과

- 시리즈 전용 라우트와 UI, 테스트, 스타일 토큰, 감사 규칙이 사라진다.
- 글 메타데이터 모델이 실제 운영하는 콘텐츠 형식과 일치한다.
- 과거 `/series` URL은 더 이상 시리즈 필터로 위장해 동작하지 않는다.
- 시리즈 글을 다시 운영하려면 라우트와 메타데이터 계약을 새 ADR에서 다시
  결정해야 한다.

## 검토한 대안

- 라우트만 숨기고 모델과 메타데이터를 유지한다: 사용처 없는 코드와 타입이
  남아 기능 폐기의 목적을 달성하지 못한다.
- 시리즈를 Engineering 필터로만 유지한다: 현재 콘텐츠가 없어 빈 탐색 경로와
  레거시 query 계약을 계속 지원해야 한다.
- 시리즈 기능을 완전히 제거한다: 현재 사용 현황에 맞고 유지 범위가 가장
  작으므로 이 대안을 선택한다.

## 검증

- 시리즈 전용 단위·컴포넌트 테스트와 E2E 시나리오를 제거한다.
- 사이트맵에 시리즈 URL이 생성되지 않는 회귀 테스트를 추가한다.
- `npm run lint`, 관련 Vitest, `npm run build`, `npm run verify:docs`를 실행한다.
