# 0019. 홈에는 에디토리얼 소개 영역과 필터 가능한 아카이브를 함께 둔다

Date: 2026-07-15
Status: Accepted

## 배경

`thebrowser.company`와 Zerolog는 큰 여백, 강한 타이포그래피 위계, 화면 가장자리에
놓인 작은 탐색 단위를 통해 개인 브랜드를 먼저 전달한다. 반면 Ark의 홈은 최신 글을
빠르게 찾는 피드 역할도 해야 하므로, 이 레이아웃을 그대로 적용하면 글 탐색성이
낮아질 수 있다.

## 결정

- 홈 상단에 사이트 정체성, 짧은 선언문, 최신 글, 아카이브·이력서·RSS 링크를 담은
  에디토리얼 소개 영역을 둔다.
- 소개 영역에만 따뜻한 중성 캔버스와 작은 포인트 모션을 사용한다. 포스트 목록은
  기존의 카테고리와 정렬 필터, list 카드 구조를 유지한다.
- `site/home`에 홈 전용 UI를 두고, 재사용 가능한 `PostList`와 블로그 도메인의
  필터링 모델은 변경하지 않는다.
- 캔버스 색상은 token으로 정의하고, 다크 테마에서도 별도로 재정의한다.

## 결과

- 방문자는 첫 화면에서 Ark의 성격과 최신 기록을 동시에 파악할 수 있다.
- 기존 피드 탐색, 인기순 정렬, 접근성 있는 키보드 포커스 흐름은 그대로 유지된다.
- 다른 목록 화면까지 포트폴리오식 레이아웃으로 바뀌는 것을 막아 도메인별 UI 책임을
  유지한다.

## 검토한 대안

- 홈 전체를 화면 높이의 포트폴리오 랜딩으로 바꾸기: 인상은 강하지만 글 목록이
  첫 화면 밖으로 밀려 블로그의 주된 탐색 흐름을 약화한다.
- 기존 카드형 필터 박스만 다듬기: 낮은 위험으로 개선할 수 있으나, 개인 기록
  공간이라는 정체성을 전달하는 위계 변화가 부족하다.
- 모든 블로그 목록의 카드 스타일을 제거하기: 일관성은 생기지만 시리즈와 긴
  설명을 탐색하는 화면의 스캔 가능성을 낮춘다.

## 검증

- `HomeHero` 컴포넌트 테스트로 최신 글과 핵심 링크의 목적지를 검증한다.
- 홈 화면 테스트로 소개 영역에 전달되는 최신 글과 필터 컨트롤의 보존을 검증한다.
- 전체 빌드와 CSS syntax 검사를 통해 token utility와 다크 테마 CSS를 검증한다.
