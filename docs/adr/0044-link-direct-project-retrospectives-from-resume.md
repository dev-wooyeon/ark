# 0044. Resume 프로젝트에는 직접 대응하는 회고글만 연결한다

Date: 2026-07-20
Status: Accepted

## 배경

공개 Resume의 한 줄 프로젝트 설명은 빠르게 훑기 좋지만, 설계 판단과 검증 근거를
확인하려면 더 긴 기록이 필요하다.

프로젝트와 관련 있어 보이는 모든 글을 연결하면 Resume의 요약 흐름이 흐려지고,
관계가 약한 글은 경력의 증거처럼 오해될 수 있다.

## 결정

- 공개 상태이며 Resume 프로젝트와 직접 대응하는 회고글만 프로젝트의 간략 소개
  아래에 내부 링크로 둔다.
- 회고글이 한 편인 경력 프로젝트는 `회고`라는 짧은 링크를 사용한다.
- 구축과 성능 개선처럼 같은 개인 프로젝트의 다른 판단을 다룬 두 글은 각각
  `구축기`, `성능개선기`로 구분한다.
- 직접 대응 글이 없는 에러 분석 자동화에는 관련 주제 글을 억지로 연결하지
  않는다.

## 결과

- 관심 있는 독자는 Resume의 문맥을 유지한 채 근거가 되는 설계 기록으로 이동할
  수 있다.
- 프로젝트 링크는 증거를 보강하지만, 경력 요약의 밀도와 읽는 속도는 유지한다.
- 연결 기준이 명확해 이후 새 글을 Resume에 추가할 때도 관계를 검토할 수 있다.

## 검토한 대안

- Resume에서 모든 글 링크를 제거한다: 가장 간결하지만 검증 근거를 제공하지
  못한다.
- 관련 주제 글까지 모두 연결한다: 탐색성은 높지만 프로젝트와의 관계가
  불분명해진다.
- 직접 대응하는 회고글만 짧게 연결한다: 가장 작은 표면적으로 근거를 제공한다.

## 검증

- ResumePage 컴포넌트 테스트로 회고 링크 다섯 개와 개인 프로젝트 링크를
  검증한다.
- Playwright로 데스크톱 Resume에서 회고 링크 다섯 개를 검증한다.
