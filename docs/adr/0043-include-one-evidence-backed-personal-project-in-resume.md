# 0043. Resume에는 증거가 있는 개인 프로젝트 하나만 제공한다

Date: 2026-07-20
Status: Accepted

## 배경

공개 Resume은 경력 요약만으로도 역할을 설명할 수 있지만, 회사 밖에서도 데이터
플랫폼을 설계하고 검증하는 방식을 보여줄 근거가 필요했다.

개인 프로젝트를 모두 나열하면 경력 요약의 밀도가 낮아지고, 제목만 적으면 실제
설계 판단을 확인하기 어렵다.

## 결정

- 개인 프로젝트는 경력 아래에 하나만 추가한다.
- `실시간 CTR 집계 파이프라인`은 경력 프로젝트와 같은 4열 문법으로 제목, 간략
  소개, 상세 설명을 제공한다.
- 프로젝트 상세에는 Kafka, Flink, ClickHouse와 Materialized View를 사용해
  조회 구조를 단순화한 판단을 남긴다.
- 구축기와 성능개선기는 같은 Resume 안에서 내부 링크로 연결한다.

## 결과

- 데이터 플랫폼 역량이 기술 나열이 아니라 설계와 검증 기록으로 이어진다.
- 개인 프로젝트 수를 제한해 공개 Resume의 빠른 훑기 흐름을 유지한다.
- 관심 있는 독자는 Resume을 벗어나지 않고 관련 회고글로 이동할 수 있다.

## 검토한 대안

- 개인 프로젝트를 넣지 않는다: 경력 중심성은 높지만 개인 기술 검증 근거가 없다.
- 모든 개인 프로젝트를 나열한다: 범위는 넓지만 Resume의 밀도가 낮아진다.
- 검증 가능한 프로젝트 하나와 관련 글만 제공한다: 가장 작은 범위에서 근거를
  제공한다.

## 검증

- ResumePage 컴포넌트 테스트로 개인 프로젝트와 두 회고글 링크를 검증한다.
- Playwright로 데스크톱 Resume에서 개인 프로젝트와 구축기 링크를 검증한다.
